Why Reports?

Quarto documents let you combine code, results, and narrative text in a single file that can be rendered to HTML, PDF, or Word. This means your analysis and its documentation live together — when your code changes, your report updates automatically. This is a core reproducibility practice.

Convert README.txt to a Quarto Document

  1. Navigate to your README.txt file in your ARCS project folder (PC: Windows Explorer; Mac: Finder)

  2. Right click on the file and select Rename

  3. Change the file extension from .txt to .qmd. You should see the icon next to the file name change.

  4. Check to see if required packages are installed by running this block of code in the console:

if (!require("knitr")) install.packages("knitr")
if (!require("rmarkdown")) install.packages("rmarkdown")
if (!require("yaml")) install.packages("yaml")
if (!require("dplyr")) install.packages("dplyr")
if (!require("stringr")) install.packages("stringr")
  1. Check if Quarto is installed in your environment by going to your terminal and entering
quarto --version

If you get a version number, that means it’s installed! If not, install from here: https://quarto.org/docs/get-started/

  1. Create a new file in your project root directory called _quarto.yml. Open it, then paste the lines below and save:
project:
  type: default

This gives Quarto a point of reference in your project.

  1. Go to your README.qmd file and add triple backticks ( ``` ) before and after your README’s file structure tree to format it as a code block

  2. In your terminal, enter

quarto render
  1. In your terminal, enter
quarto preview readme.qmd

A browser window should open and display your README.qmd file as HTML

NoteWhat is localhost?

Notice that the browser window’s address starts with localhost followed by a port number. This means your computer is serving your .qmd file locally to itself — the file is not on the internet, it is being previewed from your own machine.

Automate README.qmd Updates

Instead of manually updating your README every time your project structure changes, we can write an R code chunk that generates the file tree automatically each time you render.

  1. Check whether you have the fs package installed:
library(fs)

If you get an error, install it first:

install.packages("fs")
  1. Copy the script below:
#| echo: false
#| message: false
#| results: asis

library(fs)

# Get the actual structure
tree_output <- capture.output(dir_tree(".", recurse = 2))

# Define comments for specific files/folders
comments <- list(
  "raw_data"       = " (original data goes here)",
  "processed_data" = " (clean data goes here)",
  "scripts"        = " (code goes here)",
  "outputs"        = " (results and charts go here)",
  "README.qmd"     = " (project documentation)"
)

# Add comments to matching lines
for(item in seq_along(tree_output)) {
  line <- tree_output[item]
  for(name in names(comments)) {
    if(grepl(name, line)) {
      tree_output[item] <- paste0(line, comments[[name]])
      break
    }
  }
}

# Print the result
cat("```\n")
cat(paste(tree_output, collapse = "\n"))
cat("\n```\n")
NoteWhat this script does
  • dir_tree() generates a visual tree of your project folder up to 2 levels deep
  • capture.output() stores that tree as a character vector
  • The comments list defines annotations to add next to specific folders or files
  • The for loop finds matching lines and appends the annotations
  • cat() prints the result as a formatted code block in your rendered document

  1. Paste this script into your README.qmd file in between the triple backticks where your file tree currently appears (it’s ok to paste over the directory tree)

  2. Next to the opening triple backticks, add {r} to make it an R code chunk — it should look like this: ```{r}

  3. Save your README.qmd file

  4. In your terminal, enter quarto render then quarto preview readme.qmd

  5. Go back to your browser window to see the result. Make any changes you would like to the comments <- list() section and re-render to see them reflected

Create a Quarto Document for Publishing

In this exercise you will create a publication-quality document using a PLOS journal template, run a gene variability analysis, and generate a heatmap figure.

Setup

  1. Check whether you have the following packages installed:
library(tidyverse)
library(readxl)
library(scales)

If any are missing, install them in your console:

install.packages("tidyverse")
install.packages("readxl")
install.packages("scales")
TipChecking installed packages in Positron

You can also check in the Extensions pane by searching for the package name.

Install the PLOS Template

  1. In your Terminal tab, run the following commands one at a time:
quarto add quarto-journals/plos
quarto use template quarto-journals/plos
NoteWhat this does

This installs a Quarto extension that formats your document to match PLOS journal submission requirements — including layout, fonts, and figure formatting. This is the same approach used for many other journal templates available through quarto-journals.

Gene Variability Analysis

  1. Copy the code below and paste it into your publication.qmd file underneath the heading ## Gene Variability Analysis. Start your code block with ```{r} and close it with ```.
NoteAbout the #| lines

These are chunk options — they tell Quarto how to handle the output when this code runs inside a .qmd file. message: false suppresses package loading messages, and warning: false suppresses warnings.

#| message: false
#| warning: false

# Prepare data - rename the first column and use both ID columns
gene_data_long <- gene_data %>%
  rename(ensembl_id = `...1`) %>%
  filter(!is.na(gene_symbol), gene_symbol != "") %>%
  pivot_longer(cols = -c(ensembl_id, gene_symbol),
               names_to = "sample",
               values_to = "expression")

# Calculate variance for each gene across all samples
gene_variability <- gene_data_long %>%
  group_by(gene_symbol, ensembl_id) %>%
  summarise(
    mean_expression        = mean(expression, na.rm = TRUE),
    variance               = var(expression, na.rm = TRUE),
    coefficient_of_variation = sd(expression, na.rm = TRUE) / mean_expression,
    .groups = 'drop'
  )

# Identify top 15 most variable genes
top_genes <- gene_variability %>%
  arrange(desc(variance)) %>%
  slice_head(n = 15)

# Display summary statistics
cat("Total genes analyzed:", nrow(gene_variability), "\n")
cat("Mean variance across all genes:",
    round(mean(gene_variability$variance, na.rm = TRUE), 2), "\n")
cat("Most variable gene:", top_genes$gene_symbol[1],
    "with variance =", round(top_genes$variance[1], 2), "\n")

# Show the top 10 most variable genes
cat("\nTop 10 most variable genes:\n")
print(top_genes %>%
        select(gene_symbol, variance) %>%
        slice_head(n = 10) %>%
        mutate(variance = round(variance, 2)))

Data Visualization

  1. Copy the code below and paste it into your publication.qmd file underneath the heading ## Data Visualization. Start your code block with ```{r} and close it with ```.
#| fig-cap: "Expression heatmap showing scaled expression levels of the
#|   15 most variable genes across all samples. Blue indicates
#|   below-average expression, white indicates average expression,
#|   and red indicates above-average expression for each gene."
#| fig-width: 10
#| fig-height: 6

# Prepare heatmap data
heatmap_data <- gene_data_long %>%
  filter(gene_symbol %in% top_genes$gene_symbol) %>%
  left_join(top_genes %>% select(gene_symbol, variance),
            by = "gene_symbol") %>%
  group_by(gene_symbol) %>%
  mutate(scaled_expr = scale(expression)[, 1]) %>%
  ungroup()

# Create heatmap visualization
expression_heatmap <- heatmap_data %>%
  ggplot(aes(x = sample,
             y = reorder(gene_symbol, variance),
             fill = scaled_expr)) +
  geom_tile(color = "white", linewidth = 0.1) +
  scale_fill_gradient2(
    low   = "blue",
    mid   = "white",
    high  = "red",
    name  = "Scaled\nExpression",
    limits = c(-3, 3),
    oob   = scales::squish
  ) +
  labs(
    title    = "Expression Variability Heatmap: Top 15 Most Variable Genes",
    subtitle = "Genes ordered by variance (highest at top)",
    x        = "Sample ID",
    y        = "Gene Symbol"
  ) +
  theme_minimal() +
  theme(
    axis.text.x  = element_text(angle = 45, hjust = 1, size = 8),
    axis.text.y  = element_text(size = 10, face = "italic"),
    plot.title   = element_text(hjust = 0.5, face = "bold", size = 14),
    plot.subtitle = element_text(hjust = 0.5, size = 12),
    legend.position = "right",
    panel.grid   = element_blank()
  )

print(expression_heatmap)

# Save high-resolution figure
ggsave(paste0("outputs/", timestamp, "_expression_heatmap.png"),
       plot   = expression_heatmap,
       width  = 10,
       height = 6,
       dpi    = 300)
  1. Render your publication.qmd document to see your PLOS-formatted article
TipSaving your figure

Notice the ggsave() call at the end of the visualization block. This saves a high-resolution .png to your outputs/ folder with a timestamp in the filename — the same automated file naming pattern introduced in the Project Management section.
